### 架构澄清：「年」与 Claude 的关系

**常见误解**："简单任务年自己做，复杂任务给 Claude。"

**实际情况**：「年」本身没有智能，它已经是 Claude（或 Kimi）的包装层。所有回复都是 Claude 生成的，「年」只负责：
1. 组装 prompt（人设 + 工具描述 + 记忆上下文）
2. 调用 Claude API
3. 解析工具调用 JSON（当前是手动的，改进后由 Claude 原生处理）
4. 执行工具、记录记忆

**正确表述**：
> 「年」始终运行在 Claude 之上。改进方向是：让 Claude 更可靠地调用工具（原生 function calling），同时保留「年」的人设、记忆和本地文件操作能力。如果未来要降低成本，可以引入本地小模型处理简单对话，复杂任务再路由给 Claude。

| **定时任务** | Cron + Heartbeat（主动提醒） | ❌ 只能被动响应 |
| **语音** | TTS(edge-tts/kokoro) + ASR(SenseVoice) | ❌ 纯文本 |
| **技能系统** | 动态加载 Skills | ❌ 硬编码工具类 |
| **多端** | Web + QQ + Telegram | ❌ 只有 Electron |
| **图片理解** | v1.0.2 已支持 | ❌ 没有 |
| **记忆** | 长短期记忆 | ✅ 四层记忆更完善 |
| **代码安全** | 有 | ✅ 沙箱更健全 |

### 最值得借鉴的 5 个设计

#### 1. 三层架构（最重要）

EchoBot 把对话分成三层，解决了「年」最大的 token 浪费问题：

```
用户输入 → 决策层（意图识别）→ [闲聊/情感] → 角色层（轻量回复，无工具）
                                      ↓
                                   [任务] → Agent层（工具调用，全能力）
```

**当前「年」的问题**：你说"早安"，prompt 里依然塞了 file_tool/code_tool/http_tool/data_tool 的完整描述，浪费大量 token。

**借鉴方案**：
- 短期：前端加**路由模式切换**（Auto/Chat Only/Force Agent）
- 中期：引入本地轻量模型（Qwen2.5-3B）做意图分类，闲聊不传递工具描述

#### 2. 路由模式（最轻量）

| 模式 | 行为 | 场景 |
|------|------|------|
| **Auto** | 自动判断是否需要工具 | 默认 |
| **Chat Only** | 纯聊天，绝不触发工具 | 深夜陪伴、情感交流 |
| **Force Agent** | 强制走工具执行 | 明确要她做事时 |

**工作量**：小（前端加一个切换按钮 + 后端根据模式过滤工具描述）

#### 3. 定时任务（Cron + Heartbeat）

让 Agent 从"被动响应"升级为"主动陪伴"：
- "博士，你已经坐了 3 小时了，起来走走"
- "该睡觉了，剩下的代码明天再写"
- "你昨天说今天要处理那份报表"

**工作量**：小（Python `APScheduler` 即可）

#### 4. 语音交互（TTS + ASR）

EchoBot 的语音方案很务实：
- **TTS**：edge-tts（微软在线，免费）/ kokoro（本地离线）
- **ASR**：SenseVoice（本地，多语言）

**工作量**：小（edge-tts 一个 HTTP 请求即可接入）

#### 5. 技能系统（Skills）

可动态加载的技能库，比硬编码工具更灵活：
```
skills/
├── echobot-development/
│   ├── SKILL.md      # LLM 可读的技能描述
│   ├── config.yaml   # 配置
│   └── tools.py      # 实现
```

**工作量**：中高（当前 4 个工具够用，可后续再做）

### 借鉴优先级

| 优先级 | 功能 | 工作量 | 效果 |
|--------|------|--------|------|
| **P0** | **路由模式**（Auto/Chat Only/Force Agent） | 小 | 立刻改善体验，减少 token 浪费 |
| **P0** | **定时任务**（APScheduler） | 小 | 让「年」能主动找你，质变 |
| **P1** | **三层架构**（意图路由+轻量模型） | 中 | 大幅减少 token，响应更快 |
| **P1** | **TTS 语音**（edge-tts） | 小 | 开口说话，陪伴感翻倍 |
| **P1** | **图片理解**（Claude Vision） | 中 | 用户传截图她就能看懂 |
| **P2** | **技能系统** | 中高 | 长期可扩展性 |
| **P2** | **QQ/Telegram 机器人** | 中 | 多端覆盖 |
